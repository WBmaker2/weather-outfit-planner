# Weather Outfit Planner 귀여운 캐릭터/소품 개선 구현 계획

작성일: 2026-05-09

## 목표

- 옷을 입히는 대상이 분명히 "초등학생이 좋아할 만한 사람 모습의 캐릭터"로 보이도록 개선한다.
- 오른쪽 옷과 준비물 버튼의 소품도 이모지 의존도를 줄이고, 앱 안에서 통일된 귀여운 그림 소품으로 보이게 한다.
- 기존 핵심 동작인 클릭/드래그 선택, 같은 버튼 재클릭 해제, 캐릭터 의상 실시간 변경, 외출하기 점검을 유지한다.
- 수업용 앱답게 접근성 이름과 테스트 가능한 상태를 유지한다.

## 교체 가능성 판단

현재 구조에서는 캐릭터와 착용 소품이 `src/components/CharacterStage.tsx`의 inline SVG 레이어로 구성되어 있고, 옷장 소품은 `src/components/WardrobeItem.tsx`에서 `outfitItems.icon` 이모지를 표시한다.

따라서 교체 가능성은 높다.

1. **inline SVG 컴포넌트 방식**
   - 현재 앱 구조와 가장 잘 맞는다.
   - 추가 이미지 로딩, 경로, GitHub Pages asset 문제를 만들지 않는다.
   - `data-testid`, `aria-label`, 색상 CSS를 그대로 테스트할 수 있다.
   - 이번 1차 구현에 적용한다.

2. **정적 SVG/PNG 에셋 방식**
   - 디자이너가 만든 그림을 그대로 넣을 수 있어 완성도는 높다.
   - 파일 관리, 크기 최적화, 반응형 크롭, 접근성 대체 텍스트 관리가 추가로 필요하다.
   - 다음 단계에서 썸네일/브랜드 일관성까지 챙길 때 적합하다.

3. **AI 생성 캐릭터/소품 이미지 방식**
   - 더 그림책 같은 분위기는 만들 수 있다.
   - 하지만 소품을 선택할 때마다 옷 레이어가 정확히 맞물려야 하므로 PNG 한 장 교체보다 레이어형 SVG가 더 안정적이다.
   - 도입한다면 캐릭터 원본과 각 소품 레이어를 같은 좌표계로 생성해야 한다.

## 이번 구현 범위

1. 캐릭터 SVG를 더 둥글고 귀여운 초등학생 캐릭터로 다듬는다.
   - 큰 눈, 볼터치, 둥근 머리, 머리핀/앞머리, 작은 책가방 느낌의 기본 실루엣을 추가한다.
   - 기존 착용 레이어 `data-testid`는 유지한다.

2. 옷장 버튼의 소품을 커스텀 SVG 일러스트로 교체한다.
   - `ItemIllustration` 컴포넌트를 새로 만들어 우산, 장화, 바람막이, 반팔, 모자, 물병, 패딩, 목도리, 장갑, 샌들, 마스크, 얇은 겉옷을 통일된 스타일로 표현한다.
   - 버튼의 접근성 이름은 기존처럼 아이템 라벨만 읽히게 유지한다.

3. 착용 칩도 같은 소품 일러스트를 사용한다.
   - "우산 빼기"처럼 기존 조작 이름은 유지한다.
   - 작은 화면에서 텍스트가 밀리지 않게 아이콘 크기와 버튼 높이를 고정한다.

4. 운영 안정성도 같이 반영한다.
   - GitHub Pages workflow에서 오래된 run 취소를 켠다.
   - CI에 Playwright E2E를 추가한다.

## 검증 계획

- `npm test`
- `npm run build`
- `npm run test:e2e`
- 브라우저에서 캐릭터와 소품 시각 확인
- 주요 확인 항목:
  - 우산/장화/바람막이를 누르면 캐릭터에 즉시 보인다.
  - 같은 버튼을 다시 누르면 캐릭터에서 사라진다.
  - 옷장 버튼의 accessible name이 라벨 중심으로 유지된다.
  - 모바일 폭에서도 소품 버튼과 캐릭터가 겹치지 않는다.
